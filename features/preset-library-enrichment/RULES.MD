# Rules — Preset Library Enrichment

- Follow `stabilize/RULES.MD` where applicable.
- Plan before implementation.
- Preserve app design language and existing interactions.
- Prefer data-driven preset enrichment before preview component rewrites.
- No raw volume padding.
- No unrelated architecture rewrite.
- No new dependencies without approval.
- Do not touch sensitive areas: auth, API keys, database, export/render pipeline, AI tools, Rust/WASM, WebGPU/WGSL.
- Update `MAINTENANCE.MD` after each meaningful step.
- Update `STATE.MD` only after review/verification.
