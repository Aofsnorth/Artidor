# Transition preview variety

## Goal
Make the Transitions catalog thumbnails more varied and realistic, and less cartoonish/rainbow.

## Root cause / current state
- `procedural-preview.ts` only supplies 4 real photos in `TRANSITION_PHOTOS`, so every transition type maps to one of a tiny handful of images.
- 60 additional resized Unsplash photos (`0.jpg`–`59.jpg`) already exist in `apps/web/public/assets/transition-previews/` but are unused.
- The existing 4 webp files are already documented in `README.md` under the Unsplash License.

## Plan
1. Update `TRANSITION_PHOTOS` in `procedural-preview.ts` to include the 60 existing `*.jpg` images in addition to the 4 documented webp images.
2. Add a `TransitionPhoto` type so the generated entries are strongly typed.
3. Keep deterministic hashing and Unsplash license metadata so existing tests keep passing.
4. Subtly reduce the color-tint overlay in `transitions.tsx` so the underlying real photos are visible and not washed in accent colors.
5. Update `README.md` with a generic note for the 60 downloaded jpgs (Unsplash License).

## Files to touch
- `apps/web/src/components/editor/panels/assets/views/components/procedural-preview.ts`
- `apps/web/src/components/editor/panels/assets/views/transitions.tsx`
- `apps/web/public/assets/transition-previews/README.md`

## Risks
- The 60 jpgs were generated by an earlier ad-hoc script; we do not have per-file photographer attribution. We will rely on the Unsplash License, which does not require attribution.
- Adding 60 files to the git tree increases repo size by ~1.1 MB.
